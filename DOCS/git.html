1. Introduction to Git

Git is a distributed version control system used to track changes in code, recover old versions, and collaborate with other developers.

Git tracks changes

GitHub stores Git repositories online

Repository is your project folder tracked by Git

Git helps you work safely because every commit saves a snapshot of your project.

2. Installing Git

Install Git from the official Git website, then verify it in terminal or command prompt.

git --version
If the version number appears, Git is installed correctly.

3. Git Configuration

Set your name and email once so Git can attach them to every commit.

git config --global user.name "Shubham"
git config --global user.email "youremail@gmail.com"
git config --list

This information is stored in your Git configuration.

4. Repository

A repository is a folder where Git tracks files and history.

Type Meaning
Local Repository Repository on your own computer
Remote Repository Repository stored online on GitHub or another service

5. Git Init

Use git init to create a new Git repository in a folder.

mkdir project
cd project
git init
After initialization, Git starts tracking that folder.

6. Git Clone

Clone copies an existing repository from GitHub to your computer.

git clone https://github.com/user/repo.git
Cloning is useful when you want to work on an existing project.

7. Git Status

Git status shows the current state of your repository.

git status
Shows Meaning
Untracked files New files not added yet
Modified files Files changed after last commit
Staged files Files ready for commit

8. Git Add

Git add moves files to the staging area.

git add index.html
git add .
git add css/
Staging means you are preparing files for the next commit.

9. Git Commit

Commit saves a snapshot of your staged changes.

git commit -m "Initial commit"
Write clear commit messages so you can understand changes later.

10. Commit History

Use history commands to see older commits and project changes.

git log
git log --oneline
git log --graph
Command Use
git log Shows full commit history
git log --oneline Compact commit history
git log --graph Shows branch tree visually in terminal

11. Branches

Branches let you work on a new feature without changing the main code directly.

git branch
git branch feature/navbar
git switch feature/navbar
git switch -c feature/footer
Create branches for features, fixes, experiments, and collaboration.

12. Merge

Merge combines changes from one branch into another branch.

git switch main
git merge feature/navbar
Merge conflicts can happen if two branches change the same part of the same file.

13. Remote Repository

Remote repository connects your local project to GitHub.

git remote add origin https://github.com/user/repo.git
git remote -v
Command Use
git remote add origin Adds remote link
git remote -v Shows remote URLs

14. Push

Push uploads your commits from local repository to GitHub.

git push origin main
Push after commit when you want your changes saved online.

15. Pull

Pull downloads latest changes from the remote repository and merges them.

git pull origin main
Pull is commonly used before starting work to stay updated.

16. Fetch

Fetch downloads changes from remote without merging them immediately.

git fetch
Use fetch when you want to check updates first.

17. Stash

Stash temporarily saves unfinished work without committing it.

git stash
git stash list
git stash apply
Stash is helpful when you need to switch branches quickly.

18. Reset

Reset moves the HEAD pointer to a previous commit.

git reset HEAD~
git reset --hard HEAD~
--hard can remove changes permanently, so use it carefully.

19. Revert

Revert creates a new commit that undoes the effect of a previous commit.

git revert COMMIT_ID
Revert is safer than reset for shared repositories.

20. .gitignore

The .gitignore file tells Git which files or folders to ignore.

node_modules/
.env
dist/
*.log
Use Example
Ignore folders node_modules/
Ignore environment files .env
Ignore logs *.log

21. Git Workflow

A simple Git workflow for daily development.

1. git init
2. git add .
3. git commit -m "message"
4. git push origin main
5. git pull before new changes
A clean workflow keeps your project safe and organized.

22. Git Best Practices

  • Commit frequently.
  • Write meaningful commit messages.
  • Use branches for features.
  • Pull before pushing.
  • Use .gitignore properly.
  • Do not commit sensitive data.
  • Keep repository clean.
  • Review changes before committing.

23. Important Git Interview Questions

Question Answer
What is Git? Git is a distributed version control system.
What is a commit? A saved snapshot of the project.
What is the difference between pull and fetch? Pull downloads and merges, fetch only downloads.
What is a branch? An independent line of development.
What is a merge conflict? Conflict when two changes affect the same file section.
Learn repository, staging, commits, branches, merge, remote, push, pull, fetch, reset, revert, and .gitignore.

24. Full Git Example

This is a simple example of a complete Git workflow.

mkdir portfolio
cd portfolio
git init
git add .
git commit -m "Initial commit"
git branch dev
git switch dev
git add .
git commit -m "Added CSS"
git switch main
git merge dev
git remote add origin https://github.com/user/portfolio.git
git push -u origin main

Quick recap: Git tracks changes, GitHub stores repositories online, and branches help you work safely on new features.
