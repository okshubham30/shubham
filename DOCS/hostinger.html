1. Introduction to Hostinger

Hostinger is a web hosting platform used to host websites, manage domains, create email accounts, install WordPress, and control website files through its dashboard. The main control center is hPanel.

Web hosting keeps your website files online.

Domain is the website address.

hPanel is Hostinger’s control panel for managing your services.

Hostinger’s help center currently organizes major tasks around hPanel, DNS, File Manager, backups, SSL, email, and website setup.

2. Hosting Basics

Before building a site, you need hosting space. Your website files, database, images, and scripts are stored on a server. When someone opens your domain, the server sends your site to their browser.

Term Meaning
Hosting Online server space for website files
Server Computer that delivers your website
Bandwidth Data transferred between server and visitors
Storage Space for files, images, and database

Shared Hosting

Good for beginner websites, blogs, and small projects.

WordPress Hosting

Optimized for WordPress sites and easy installation.

VPS Hosting

More control and power for advanced websites or applications.

3. hPanel Overview

hPanel is the Hostinger dashboard where you manage websites, domains, email, databases, backups, files, and security settings. 

hPanel
├─ Websites

Area Use

Domains Manage domain names and DNS
Files Upload and edit website files
Backups Restore old versions of your website

4. Domain Basics

A domain is the name people type in a browser, like example.com. It points to your hosting server through DNS records.

Part Meaning
Domain name Website address
TLD Extension like .com, .in, .net
Subdomain Extra part like blog.example.com
Nameserver Tells where DNS is managed
If the domain is not pointing correctly, the site may not open even if hosting is active.

5. DNS Management

DNS connects your domain to the correct server. In Hostinger, DNS settings are available from the Domains area in hPanel. 

Important DNS records:
- A record
- CNAME record
- MX record
- TXT record
- NS record
Record Use
A Points domain to an IP address
CNAME Points one name to another name
MX Handles email delivery
TXT Used for verification and email security
NS Nameservers for DNS control

6. Create a Website

A website can be created in Hostinger by using WordPress, uploading custom files, or using the Website Builder.

Custom HTML Website

Best for simple static websites built with HTML, CSS, and JavaScript.

WordPress Site

Good for blogs, business sites, and dynamic content.

Website Builder

Drag-and-drop option for quick website creation.

Basic flow:
1. Buy hosting
2. Connect domain
3. Upload files or install WordPress
4. Add SSL
5. Publish your website

7. WordPress on Hostinger

WordPress is one of the most common ways to build websites on Hostinger. It lets you create pages, blog posts, menus, plugins, and themes without coding everything from scratch.

WordPress Part Meaning
Theme Controls site design
Plugin Adds extra features
Post Blog article
Page Static content like About or Contact

WordPress setup tip: install WordPress first, then choose a theme, add pages, and configure menus.

8. Hostinger Website Builder

The Website Builder is useful when you want a visual editor instead of coding. It is simple for landing pages, business websites, and quick portfolio sites.

  • Drag and drop design
  • Ready-made templates
  • Simple section editing
  • Fast publishing workflow
Use the builder when speed matters more than custom coding control.

9. File Manager

File Manager is where you upload, edit, rename, and delete website files. For custom websites, the main files are usually inside the public directory. 

Typical web root files:
index.html
style.css
script.js
assets/
images/
DOCS/
File Manager Task Use
Upload Add files to your website
Edit Change code directly
Rename Change file or folder name
Delete Remove unwanted files
Always keep a backup before deleting or replacing live website files.

10. Uploading Website Files

If your site is made in HTML, CSS, and JavaScript, you can upload the files manually using File Manager or FTP. 

Example upload flow:
1. Open hPanel
2. Go to Websites
3. Open Dashboard
4. Open File Manager
5. Upload ZIP or individual files
6. Extract ZIP if needed
7. Make sure index.html is in the root folder
Common Mistake Result
index.html is inside a folder Website may not open correctly
Wrong file name Server may show default page
Broken paths CSS or images may not load

11. Database Basics

Databases store dynamic website data such as users, posts, product information, and form submissions. WordPress usually uses MySQL or MariaDB.

Term Meaning
Database Structured data storage
Table Collection of rows and columns
Row One record
Column Field in a table

Static sites may not need a database, but WordPress and many apps do.

12. SSL Certificate

SSL protects the connection between the visitor and your website. It also helps with trust and browser security warnings. 

HTTP  ->  not encrypted
HTTPS ->  encrypted with SSL
Feature Use
HTTPS Secure website connection
Padlock icon Shows secure status in browser
Mixed content HTTPS page loading HTTP resources
After enabling SSL, update internal links and images so everything loads through HTTPS.

13. Email Setup

Hostinger also supports email-related setup through hPanel. You can create professional email addresses using your domain name. 

Example:
info@yourdomain.com
support@yourdomain.com
admin@yourdomain.com
Email Part Meaning
Mailbox Email account inbox
SMTP Sends emails
IMAP Syncs mail with server
POP3 Downloads mail to device

14. FTP / SFTP

FTP and SFTP are file transfer methods used to upload or manage website files from an external file manager or code editor.

Protocol Use
FTP Basic file transfer
SFTP Secure file transfer
Use SFTP when possible because it is more secure than plain FTP.

15. PHP and Website Development

PHP is often used for dynamic websites on Hostinger. It can handle forms, logins, dashboard pages, and database connections. 

<?php
echo "Hello from PHP";
?>
Use Case Example
Form handling Contact form submission

16. Security Basics

Website security means protecting files, passwords, forms, and user data from misuse.

  • Use strong passwords
  • Enable SSL
  • Keep plugins and themes updated
  • Use secure file permissions
  • Take regular backups
Small security habits prevent big website problems later.

17. Backups

Backups are saved copies of your website files and databases. They help you recover from mistakes, broken updates, or accidental deletions.

Backup Type What it saves
Files backup HTML, CSS, JS, images, uploads
Database backup WordPress data or app data
Full backup Files + database together
Backup before changing themes, moving files, or editing live code.

18. Website Migration

Migration means moving a website from another host to Hostinger. Usually this involves copying files, exporting the database, and connecting the domain. 

Migration steps:
1. Download website files
2. Export database
3. Upload files to Hostinger
4. Import database
5. Update configuration
6. Point domain to Hostinger
7. Test the site
Step Goal
Files Move code and assets
Database Move dynamic content
DNS Send domain to new server

19. SEO and Speed

SEO helps search engines understand your website, while speed improves user experience and page performance.

Topic Good Practice
SEO title Clear and keyword-focused
Meta description Short summary of page content
Image size Keep images optimized
Speed Minimize heavy files and scripts

Fast websites usually feel better to users and are easier to manage.

20. Common Troubleshooting

If the website does not work properly, check these common issues first.

Problem Possible Cause
Website not opening Wrong DNS or domain not pointed
CSS not loading Incorrect file path
Image missing Wrong filename or folder path
Login not working Database or PHP issue
403/404 error Permission or file location issue
Check the browser console and server file paths when debugging a broken page.
