1. Introduction to HTML

HTML means HyperText Markup Language. It is used to create the structure of web pages. HTML tells the browser what content should appear on the page.

HTML gives structure

CSS gives style

JavaScript gives behavior

HTML is not a programming language. It is a markup language used to organize content.

2. Basic HTML Structure

Every HTML file starts with the standard boilerplate.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My First Page</title>
</head>
<body>
  <h1>Hello World</h1>
</body>
</html>

<!DOCTYPE html>

Tells the browser to use HTML5 mode.

<html>

The root element of the whole page.

<head>

Contains metadata, title, icons, links, and settings.

<body>

Contains all visible content shown on the page.

3. HTML Tags, Elements, and Attributes

A tag is the opening or closing part like <p>. An element is the complete tag plus content. An attribute gives extra information.

Term Meaning Example
Tag Opening or closing marker <p>
Element Full HTML unit <p>Hello</p>
Attribute Extra information inside tag href="page.html" 
<a href="https://example.com" target="_blank">Visit Site</a>
Attributes always go inside the opening tag.

4. Headings, Paragraphs, and Text Formatting

Headings

HTML has six heading levels from <h1> to <h6>.

<h1>Main Heading</h1>
<h2>Sub Heading</h2>
<h3>Section Heading</h3>
<h4>Smaller Heading</h4>
<h5>Even Smaller</h5>
<h6>Smallest Heading</h6>

Paragraphs and text styles

<p>This is a paragraph.</p>
<strong>Bold text</strong>
<em>Italic text</em>
<mark>Highlighted text</mark>
<small>Small text</small>
<del>Deleted text</del>
<ins>Inserted text</ins>

<strong> means important bold text.

<em> means emphasis and usually appears italic.

5. Links

Links connect one page to another page, a website, or a section inside the same page. 

<a href="about.html">Go to About Page</a>
<a href="https://google.com" target="_blank">Open Google</a>
<a href="#contact">Go to Contact Section</a>

href

Defines the destination link address.

target="_blank"

Opens the link in a new tab.

6. Images

Images are inserted with <img>. It is a self-closing tag. 

<img src="photo.jpg" alt="My Photo" width="300" />
Always use alt text. It helps accessibility and SEO.

7. Lists

HTML supports ordered lists, unordered lists, and description lists.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<ol>
  <li>Open file</li>
  <li>Write code</li>
  <li>Save file</li>
</ol>
List Type Tag Use
Unordered list <ul> Bullets
Ordered list <ol> Numbers
Description list <dl> Term and meaning

8. Tables

Tables are used for rows and columns. They are useful for structured data.

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>Shubham</td>
    <td>20</td>
  </tr>
</table>
Tag Purpose
<table> Table container
<tr> Table row
<th> Table header
<td> Table data cell

9. Forms

Forms collect user input like name, email, password, and messages.

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" />

  <label for="msg">Message:</label>
  <textarea id="msg" name="msg"></textarea>

  <button type="submit">Send</button>
</form>

<label> is used to describe the input.

required makes a field compulsory.

action is where form data goes.

method is usually GET or POST.

Use proper labels for better accessibility and better user experience.

10. Semantic HTML

Semantic tags clearly tell the meaning of content. They improve SEO, accessibility, and code readability. 

<header>Top section</header>
<nav>Navigation links</nav>
<main>Main content</main>
<section>Thematic section</section>
<article>Independent content</article>
<aside>Sidebar content</aside>
<footer>Bottom section</footer>
Tag Meaning
<header> Top area of page or section
<nav> Navigation area
<main> Main content of page
<section> Group of related content
<article> Self-contained content
<aside> Extra or side content
<footer> Bottom area of page or section

11. Block and Inline Elements

Block elements start on a new line and take full width. Inline elements stay inside a line.

Block Elements Inline Elements
div, p, h1-h6, section, article, ul, ol span, a, strong, em, img, code

Block example: <div>, <p>

Inline example: <span>, <a>

12. Media Tags

HTML can also show audio, video, and embedded content.

<audio controls>
  <source src="song.mp3" type="audio/mpeg" />
</audio>

<video controls width="400">
  <source src="video.mp4" type="video/mp4" />
</video>

<iframe src="https://example.com"></iframe>
Use media carefully because large files can slow down your page.

13. Comments

Comments are not shown on the page. They are useful for notes in code.

<!-- This is a comment -->

14. HTML Entities

Entities are used for special characters.

Entity Meaning Output
&lt; Less than <
&gt; Greater than >
&amp; Ampersand &
&nbsp; Non-breaking space space

15. Head and Meta Tags

The <head> section contains important page settings. 

<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="description" content="HTML notes" />
  <title>HTML Notes</title>
</head>

charset defines character encoding.

viewport makes the page responsive on mobile devices.

description helps search engines understand the page.

16. Accessibility Basics

Accessibility means making web pages easy to use for everyone, including users with screen readers or keyboard navigation.

Alt text

Always add alt text to images.

Labels

Use labels for form inputs.

Heading order

Use headings in a proper order.

Semantic tags

Use header, main, section, and footer.

Accessibility is not optional. It improves usability for all users.

17. HTML Best Practices

  • Use semantic tags instead of only div tags.
  • Keep heading order logical.
  • Always add alt text to images.
  • Use lowercase tag names.
  • Close tags properly.
  • Keep code readable with proper indentation.
  • Use comments only when needed.
  • Make forms clear and simple.

18. Full Example Page

This is a complete simple HTML page using many basic tags.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My Page</title>
</head>
<body>
  <header>
    <h1>My Website</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
      <a href="#contact">Contact</a>
    </nav>
  </header>

  <main>
    <section id="home">
      <h2>Welcome</h2>
      <p>This is my first web page.</p>
    </section>

    <section id="about">
      <h2>About Me</h2>
      <p>I am learning HTML.</p>
    </section>

    <section id="contact">
      <h2>Contact</h2>
      <form>
        <input type="text" placeholder="Your Name" />
        <button type="submit">Send</button>
      </form>
    </section>
  </main>

  <footer>
    <p>© 2026 Shubham</p>
  </footer>
</body>
</html>

Quick recap: HTML is the structure of the page, CSS styles it, and JavaScript makes it interactive.

19. Important Interview Points

  • HTML is a markup language, not a programming language.
  • HTML gives structure to web pages.
  • Semantic tags improve SEO and readability.
  • Always add alt text to images.
  • Forms collect user input.
  • Links are created using anchor tags.
  • The head section contains metadata and settings.
