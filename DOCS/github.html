1. Introduction to GitHub

GitHub is a cloud-based platform used to host Git repositories, collaborate with developers, manage code, review changes, and build software projects online.

GitHub stores code online

Repository is a project space on GitHub

Collaboration happens through branches, pull requests, and issues

GitHub is not Git itself. Git is the version control tool, and GitHub is a hosting and collaboration platform built around Git.

2. Git vs GitHub

Many beginners confuse Git and GitHub, but they are different.

Git GitHub
Version control system Online platform for Git repositories
Installed on your computer Used through browser or Git tools
Tracks file changes Stores projects, issues, pull requests, and more
Works locally Supports team collaboration and sharing
Git manages history. GitHub helps you share that history with others.

3. Creating a GitHub Account

To start using GitHub, create an account first.

  1. Go to GitHub in your browser.
  2. Click Sign Up.
  3. Enter email, password, and username.
  4. Verify your email address.
  5. Set up your profile picture and bio.
Important profile details include your username, bio, location, website link, and pinned repositories.

4. GitHub Dashboard

The dashboard is the home screen after login. It helps you access repositories and activity quickly.

Repositories

All your projects in one place.

Pull Requests

Review and manage code changes.

Issues

Track bugs, tasks, and feature requests.

Actions

Automate testing and deployment workflows.

5. Repository

A repository is a project folder on GitHub that stores code, files, commits, branches, and documentation.

Type Meaning
Public Repository Anyone can view it
Private Repository Only invited users can access it
Forked Repository Copy of another repository in your account
A good repository usually has a README, license, .gitignore file, and clear folder structure.

6. Creating a Repository

To create a new repository on GitHub, click the New button and fill in the details.

  1. Enter repository name.
  2. Add optional description.
  3. Select public or private.
  4. Choose whether to add README.
  5. Click Create repository.
For beginners, enabling README and .gitignore is usually helpful.

7. README.md

The README file explains what the project is, how to use it, and how to install it.

# Portfolio Website

A responsive personal portfolio built using HTML, CSS, and JavaScript.

## Features
- Responsive design
- Dark mode
- Contact section

## Installation
1. Clone the repository
2. Open index.html in browser
A strong README makes your project easy to understand for others, including recruiters and teammates.

8. GitHub Profile README

You can create a special repository with the same name as your username to show a profile README on your profile page.

username/username

What to include

Introduction, skills, projects, contact links, and achievements.

Why it matters

It helps your profile look professional and personal.

9. Clone Repository

Clone downloads a repository from GitHub to your local computer.

git clone https://github.com/user/repository.git
Cloning is useful when you want to work on a project locally.

10. Fork

Fork creates your own copy of someone else's repository inside your GitHub account.

Fork is commonly used in open source contribution when you want to modify a project without changing the original repository directly.
Fork Clone
Creates a copy on GitHub Downloads a copy to your local machine
Used for contribution Used for local development

11. Star

Star is like bookmarking a repository so you can find it later and show appreciation.

Starring a repository does not copy it. It only saves it to your starred list.

12. Watch

Watch lets you follow updates from a repository.

When you watch a repository, you can receive notifications about issues, pull requests, and new activity.

13. Branches on GitHub

Branches help teams work on different features at the same time without disturbing the main code.

git branch feature-login
git switch feature-login
git push origin feature-login
Branch Purpose
main Stable production code
dev Development work
feature/* New features
hotfix/* Urgent bug fixes

14. Pull Request

A pull request is a request to merge your changes into another branch, usually main or dev.

  1. Create a branch.
  2. Make changes and commit them.
  3. Push the branch to GitHub.
  4. Open a pull request.
  5. Review, discuss, and merge.
Pull requests are used for code review, collaboration, and quality control.

15. Issues

Issues are used to track bugs, tasks, improvements, and feature requests.

Title: Navbar not responsive
Description: Navbar breaks on mobile screens.
Issues help teams organize work and discuss problems clearly.

16. Labels

Labels help classify issues and pull requests.

Label Use
bug Something is broken
enhancement Improvement request
documentation Docs-related changes
good first issue Easy for beginners

17. Milestones

Milestones group issues and pull requests into a bigger goal, such as a release version.

Example milestone: Version 1.0 Release

18. GitHub Projects

Projects help you manage tasks in a visual board format like Kanban.

Todo

Tasks not started yet.

In Progress

Tasks currently being worked on.

Review

Tasks waiting for feedback.

Done

Completed tasks.

19. Discussions

Discussions are used for questions, ideas, and community communication.

Discussions are helpful when you want a more open conversation than issues and pull requests.

20. GitHub Actions

GitHub Actions is GitHub's automation tool for building, testing, and deploying code.

name: Build

on: push

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm install
      - run: npm test
Actions are often used for CI/CD workflows, linting, tests, and deployment.

21. GitHub Pages

GitHub Pages is a free hosting service for static websites directly from a repository.

Supports Does not support
HTML, CSS, JavaScript Server-side code like PHP or Node.js backend
GitHub Pages is great for portfolios, documentation sites, and small static projects.

22. Releases and Tags

Releases are published versions of your software. Tags mark important points in the repository history.

git tag v1.0
git push origin v1.0
Releases are useful when you want to distribute a stable version of your project.

23. Security

GitHub provides tools to keep repositories and code safer.

Security Feature Meaning
Secret Scanning Detects passwords and tokens accidentally committed
Dependabot Helps update dependencies
Code Scanning Finds security issues in code
Never commit API keys, passwords, or private credentials to public repositories.

24. Organizations and Teams

Organizations are used by companies, communities, and groups to manage multiple repositories and members.

Organizations

Central place for shared repositories.

Teams

Groups with specific permissions and responsibilities.

Permissions

Control who can read, write, or admin a repository.

25. GitHub Workflow

A simple workflow used in real projects.

1. Create repository
2. Clone repository
3. Create a new branch
4. Make changes
5. Commit changes
6. Push branch to GitHub
7. Create pull request
8. Review code
9. Merge pull request
10. Pull latest changes
This workflow keeps code organized and reduces mistakes in teamwork.

26. Open Source Contribution

GitHub is widely used for open source, where people contribute to public projects.

1. Fork repository
2. Clone your fork
3. Create branch
4. Make changes
5. Commit and push
6. Open pull request
Open source contribution is one of the most important skills for new developers to learn.

27. GitHub Best Practices

  • Write clear README files.
  • Use branches for features and fixes.
  • Open pull requests for review.
  • Use meaningful commit messages.
  • Never commit secrets or passwords.
  • Add a .gitignore file.
  • Use issues and projects to track work.
  • Keep repository names simple and clear.
  • Protect important branches like main.
  • Enable two-factor authentication for security.

28. Important GitHub Interview Questions

Question Answer
What is GitHub? GitHub is an online platform for hosting Git repositories and collaborating on code.
What is a repository? A project folder stored on GitHub.
What is a pull request? A request to merge changes from one branch into another.
What is a fork? A copy of someone else's repository in your account.
What is the difference between fork and clone? Fork creates a copy on GitHub, clone downloads a copy locally.
What is GitHub Actions? An automation service for CI/CD and workflows.
What is GitHub Pages? A free hosting service for static websites.
Learn repository, fork, clone, branch, pull request, issue, actions, pages, security, and workflow basics.

29. Full GitHub Example

This is a simple end-to-end workflow for a GitHub project.

git init
git add .
git commit -m "Initial commit"

git branch dev
git switch dev

git add .
git commit -m "Added home page"

git push origin dev

Open Pull Request on GitHub

Merge Pull Request

git switch main
git pull origin main

Quick recap: GitHub hosts repositories online, helps teams collaborate through pull requests and issues, and supports automation, hosting, and project management.
