HTML

HyperText Markup Language — the skeleton of every web page.

Chapter 01
🏗️
Basic Document Structure Every HTML page starts here
HTML 
<!-- HTML boilerplate -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport"
        content="width=device-width" />
  <title>Page Title</title>
</head>
<body>
  <h1>Hello World 🌍</h1>
</body>
</html>
💡 <!DOCTYPE html> tells the browser to use modern HTML5 standards. Always put it first.
🏷️
Semantic HTML Tags Meaningful structure for better SEO & accessibility
TagPurpose
<header>Top of the page / section
<nav>Navigation links
<main>Main unique content
<section>Thematic grouping
<article>Self-contained content
<aside>Sidebar / related info
<footer>Bottom of page / section
✅ Semantic tags help screen readers, search engines, and future developers understand your page structure.

SelectorTargets
*All elements
h1Element by tag
.classElement by class

p::beforePseudo-element
div > pDirect child

📦
The Box Model Every element is a box
CSS 
.box {
  width:   200px;
  height:  100px;

  /* Space inside the border */
  padding: 20px;

  /* The border itself */
  border:  2px solid #38bdf8;

  /* Space outside the border */
  margin:  16px;

  /* Include padding in width */
  box-sizing: border-box;
}
↔️
Flexbox One-dimensional layout system
CSS 
.container {
  display:         flex;
  flex-direction:  row;      /* or column */
  justify-content: center;   /* main axis */
  align-items:     center;   /* cross axis */
  gap:             16px;
  flex-wrap:       wrap;
}

.child {
  flex: 1;          /* grow & shrink equally */
}
💡 Use Flexbox for navigation bars, card rows, and centring elements — it's perfect for 1D layouts.
🔲
CSS Grid Two-dimensional layout system
CSS 
.grid {
  display:               grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows:    auto;
  gap:                   20px;
}

/* Span across columns */
.wide-item {
  grid-column: 1 / -1;
}
💡 Grid is perfect for full page layouts, card grids, and anything requiring both rows & columns.
🎬
Variables & Animations Custom properties & keyframes
CSS 
/* CSS Variables */
:root {
  --primary: #38bdf8;
  --gap:     16px;
}

.btn {
  color: var(--primary);
  gap:   var(--gap);
  animation: fadeIn 0.5s ease;
}

@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

Git & GitHub

Track changes, collaborate and never lose your work again.

Chapter 03
⌨️
Essential Commands The ones you use every day
TERMINAL 
# Initialize a new repository
git init

# Clone an existing repo
git clone https://github.com/user/repo.git

# Stage changes
git add .          # all files
git add index.html  # specific file

# Commit with message
git commit -m "feat: add hero section"

# Push to remote
git push origin main

# Pull latest changes
git pull
🌿
Branching & Merging Work on features without breaking main
TERMINAL 
# List all branches
git branch

# Create a new branch
git branch feature/navbar

# Switch to branch
git checkout feature/navbar
# Or shorthand (create + switch)
git checkout -b feature/navbar

# Merge branch into main
git checkout main
git merge feature/navbar

# Delete a branch
git branch -d feature/navbar
🔍
Status & History Understanding your repo state
TERMINAL 
# Check what's changed
git status

# View commit history
git log
git log --oneline     # compact view
git log --graph      # branch tree

# See what changed in a file
git diff index.html

# Unstage a file
git restore --staged index.html

# Undo last commit (keep changes)
git reset --soft HEAD~1
🐙
GitHub Flow Standard collaboration workflow

A clean workflow for working with teams on GitHub:

StepAction
1️⃣Create a branch for your feature
2️⃣Make commits on that branch
3️⃣Push branch to GitHub
4️⃣Open a Pull Request (PR)
5️⃣Code review by teammates
6️⃣Merge PR into main
7️⃣Delete the branch 🎉
💡 Never commit directly to main. Always use feature branches and Pull Requests.

Events Listening to user interactions
JAVASCRIPT 
const btn = document.querySelector("#btn");

// Click event
btn.addEventListener("click", (e) => {
  console.log("Button clicked!");
});

// Scroll event
window.addEventListener("scroll", () => {
  if (window.scrollY > 100) {
    btn.classList.add("visible");
  }
});

// Keyboard event
document.addEventListener("keydown", (e) => {
  console.log(e.key);
});
🔧
Functions & Arrays Reusable logic & data collections
JAVASCRIPT 
// Arrow function
const greet = (name) => `Hello, ${name}! 👋`;

// Array methods
const nums = [1, 2, 3, 4, 5];

nums.map(n => n * 2);      // [2,4,6,8,10]
nums.filter(n => n > 2);   // [3,4,5]
nums.reduce((a,b) => a+b); // 15
nums.forEach(n => {
  console.log(n);           // logs each
});

// Find & includes
nums.find(n => n > 3);     // 4
nums.includes(3);          // true

Responsive Design

Build layouts that look perfect on every screen size.

Chapter 05
📐
Media Queries Style by screen size
CSS 
/* Mobile first — default styles */
.container { padding: 16px; }

/* Tablet — 600px and up */
@media (min-width: 600px) {
  .container { padding: 32px; }
}

/* Desktop — 900px and up */
@media (min-width: 900px) {
  .container { padding: 5%; }
}

/* Max-width — target small screens */
@media (max-width: 480px) {
  .grid { grid-template-columns: 1fr; }
}
📏
Fluid Typography & Units Sizes that adapt automatically
UnitMeaning
pxFixed pixels — absolute
%Relative to parent element
emRelative to parent font-size
remRelative to root font-size (16px)
vw% of viewport width
vh% of viewport height
clamp()Fluid value between min & max
CSS 
font-size: clamp(16px, 4vw, 32px);
